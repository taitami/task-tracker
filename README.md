# Лабораторна робота №1: Розгортання Web-сервісу з автоматизацією

## 📌 Опис проєкту
Цей проєкт є простою веб-платформою `mywebapp`, яка реалізує сервіс відстеження задач (**Task Tracker**). Застосунок розроблений на базі Node.js/TypeScript та взаємодіє з реляційною базою даних MariaDB.

Мета роботи — практичне розгортання веб-сервісу на віртуальній машині під керуванням Linux (Ubuntu 24.04 Server), налаштування зворотного проксі-сервера (Nginx), розподіл прав користувачів, конфігурація керування через Systemd та повна автоматизація процесу інсталяції за допомогою єдиного Bash-скрипта.

## 🔢 Розрахунок варіанту індивідуального завдання
Варіант розраховано на основі порядкового номера студента (або номера залікової книжки): **$N = 16$**.

* **Тематика веб-застосунку ($V_3$):** $V_3 = (16 \pmod 3) + 1 = 1 + 1 = \mathbf{2}$ $\rightarrow$ **Task Tracker**.
* **Спосіб конфігурації та СУБД ($V_2$):** $V_2 = (16 \pmod 2) + 1 = 0 + 1 = \mathbf{1}$ $\rightarrow$ Конфігурація через **аргументи командного рядка**, база даних — **MariaDB**.
* **Порт застосунку ($V_5$):** $V_5 = (16 \pmod 5) + 1 = 1 + 1 = \mathbf{2}$ $\rightarrow$ Веб-застосунок слухає порт **`5200`**.

## 🌐 Загальна архітектура системи
Усі компоненти системи розгорнуті на одній віртуальній машині. 
Взаємодія компонентів побудована за такою схемою:
`Клієнт` $\rightarrow$ `Nginx (Рорт 80)` $\rightarrow$ `mywebapp (Порт 5200)` $\rightarrow$ `MariaDB (Порт 3306)`.

## Лабораторна робота №4: IaC (Terraform) та Configuration Management (Ansible)

Інфраструктура проєкту розгортається на двох віртуальних машинах (`worker` та `db`), створених автоматично.

### Запуск Terraform
1. Перейдіть до папки `terraform`: `cd terraform`
2. Ініціалізуйте провайдер: `terraform init`
3. Створіть ВМ: `terraform apply` (підтвердіть введенням `yes`).
4. Скопіюйте IP-адреси ВМ, які з'являться на екрані.

### Запуск Ansible
1. Перейдіть до папки `ansible`: `cd ansible`
2. Відкрийте файл `inventory.ini` та впишіть актуальні IP-адреси, які ви отримали від Terraform.
3. Запустіть налаштування командою: 
   `ansible-playbook -i inventory.ini playbook.yml`

*Усі налаштування застосовуються декларативно (використовуються модулі apt, user, systemd, template). База даних не доступна ззовні, а застосунок динамічно отримує її IP-адресу з Ansible templates.*
